# Plan 03-01: Field Resolution Polish

## Objective
Add smart field ordering, validation, director-as-sender fix, and multi-step wizards for complex documents.

## Tasks

### Task 1: Director-as-sender fix
<read_first>
- `src/index.ts` — autoFillValue function
</read_first>

<action>
In `autoFillValue`, when `context.role === 'director'`:
- `sender_name` should use `context.user.full_name` (the director is the sender)
- `sender_position` should use `context.user.position` (Direktor)
- Recepient is likely a teacher or external, so `recipient_name` / `recipient_title` don't auto-fill
</action>

<acceptance_criteria>
- Director logs in → creates ariza → sender_name shows director's name
- Director creates buyruq → director's position shows as sender
</acceptance_criteria>

### Task 3: Field validation
<read_first>
- `src/index.ts` — validate functions in prompts
</read_first>

<action>
Add validation per field name:
- `date`, `start_date`, `effective_date`, `issue_date`, `meeting_date`, `birth_date`, `week_start`, `week_end`, `replacement_date`: validate `/^\d{2}\.\d{2}\.\d{4}$/` — "Sana KK.OO.YYYY formatida bo'lishi kerak"
- Fields ending in `_phone`: validate `/^\+998\d{9}$/` — non-blocking warning "Telefon +998XXXXXXXXX formatida bo'lishi tavsiya etiladi"
- Fields containing `hours`, `percent`, `count`, `amount`, `salary`, `students`, `total_`: validate `/^\d+$/` — "Faqat raqam kiritilishi mumkin"
- Keep existing empty-field validation for all fields
</action>

<acceptance_criteria>
- Invalid date format shows Uzbek error
- Invalid phone shows warning (non-blocking)
- Non-numeric hours/percents shows error
- Empty fields blocked
</acceptance_criteria>

### Task 4: Multi-step headers for complex docs
<read_first>
- `src/index.ts` — field collection section
</read_first>

<action>
Before prompting fields for complex document types (`buyruq_asosiy`, `buyruq_kadr`, `tarif_malaka`, `pedagogik_kengash_bayoni`, `shaxsiy_delo`), print step headers:
- Buyruq types: "1-qadam: Ma'lumot" (order_title, order_number, date) → "2-qadam: Buyruq matni" (order_text, basis, appendix)
- Tarif: "1-qadam: Xodim ma'lumotlari" → "2-qadam: Hisob-kitob" → "3-qadam: Tasdiqlash"
- Bayon: "1-qadam: Majlis ma'lumotlari" → "2-qadam: Kun tartibi" → "3-qadam: Qarorlar"
- Shaxsiy delo: "1-qadam: O'quvchi ma'lumotlari" → "2-qadam: Hujjatlar"
- Other types: No step headers (default flow)
</action>

<acceptance_criteria>
- Complex docs show step headers between field groups
- Simple docs (ariza, tushuntirish xati) show no step headers
</acceptance_criteria>
