- `src/db.ts` — new types and supabase client
</read_first>

<action>
Create `src/auth.ts`:
- Use `inquirer` for prompts (Uzbek language)
- Flow:
  1. Display welcome box with "Zunoora" title
  2. Prompt "Login ID (8 symbol):" — validate non-empty
  3. Prompt "PIN kod:" — use `type: 'password'` mask input, validate 4-6 digits
  4. Hash PIN with simple approach for MVP (e.g., crypto.createHash('sha256'))
  5. Query `teachers` table: `.select('*, school_data:schools(*)').eq('login_id', id).eq('pin_hash', hashedPin).maybeSingle()`

  8. If director found → load teachers + classes → return `{ user, school, teachers, classes, role: 'director' }`
  9. If neither → "ID yoki PIN noto'g'ri" → retry
- Keep 3-retry infra error handling from existing code
- Keep existing `.maybeSingle()` pattern (distinguishes "no match" from "DB error")
- Import types from `db.ts`

Auth HTML-style comments from existing code should be preserved as code comments.
